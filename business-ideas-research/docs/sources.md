# Источники сигналов

## Стартовый слой: стабильные и доступные источники

| Источник | Что искать | Доступ | Комментарий |
| --- | --- | --- | --- |
| Hacker News | Show HN, Ask HN, Launch HN, обсуждения dev/business pain | Public API | Хорош для ранних B2B, devtools, AI и SaaS-сигналов |
| Product Hunt | Новые продукты, теги, комментарии, голоса | GraphQL API | Хорош для launch-сигналов, но важно не путать хайп и спрос |
| GitHub Trending | Новые репозитории, темы, devtools | Web/API через GitHub | Хорош для технологий до коммерциализации |
| Stack Exchange | Повторяющиеся профессиональные проблемы | Public API | Полезно для технических и операционных болей |
| YouTube | Новые обучающие видео, обзоры инструментов, спрос по нишам | Data API | Нужен API key и учет квот |
| RSS/newsletters | Новости ниш, стартапы, funding, regulations | RSS/ручной список | Надежный слой для обзоров |

## Правила работы с источниками

1. Использовать официальные API там, где они есть.
2. Не сохранять персональные данные без необходимости.
3. Не обходить ограничения платформ.
4. В карточках сохранять ссылку на источник, дату и краткую выжимку.
5. Не доверять одному источнику: хорошая идея должна иметь 2-3 независимых сигнала.
