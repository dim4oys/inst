# Pipeline: от сигнала до идеи

## 1. Collect

Собираем сырые сигналы:

- заголовок;
- текст или краткую выжимку;
- URL;
- источник;
- дата публикации;
- автор/канал, если это публичная сущность;
- метрики: голоса, комментарии, просмотры, звезды, рейтинг.

Сырые данные сохраняются в `data/raw/`.

## 2. Normalize

Приводим к единому виду:

```json
{
  "id": "source-unique-id",
  "source": "hacker_news",
  "market": "global",
  "language": "en",
  "published_at": "2026-04-25",
  "title": "Example signal",
  "summary": "Short description",
  "url": "https://example.com",
  "metrics": {
    "votes": 0,
    "comments": 0
  },
  "tags": ["ai", "saas", "workflow"],
  "pain_markers": ["manual process", "expensive"]
}
```

Нормализованные данные сохраняются в `data/normalized/`.

## 3. Cluster

Группируем сигналы по темам:

- похожая боль;
- похожая аудитория;
- похожая работа, которую нужно сделать;
- похожий тип продукта.

Пример:

```text
AI для бухгалтеров
  - жалобы на ручной ввод документов
  - новые продукты для invoice automation
  - вакансии automation consultant
  - рост YouTube-контента по теме
```

## 4. Score

Оцениваем кластер или конкретную идею по `docs/scoring.md`.

## 5. Write idea card

Если идея проходит минимальный порог, создаем Markdown-карточку в `ideas/watch/` или `ideas/hot/`.

## 6. Report

Раз в неделю создаем отчет:

- top ideas;
- растущие темы;
- слабые сигналы;
- что проверить руками;
- какие источники дали лучший результат.

## 7. Decide

Для каждой сильной идеи выбираем следующий тест:

- лендинг;
- интервью с 5-10 потенциальными клиентами;
- ручной concierge MVP;
- парсинг спроса/конкурентов;
- платная реклама на маленький бюджет;
- публикация в комьюнити.
